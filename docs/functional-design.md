# 功能设计文档

## 一、功能总览

本系统围绕"管理持仓 → 分析诊断 → 扫描市场 → 推荐操作"四个环节设计。

| 模块 | 页面 | 核心功能 |
|------|------|---------|
| 总览 | Dashboard | 资产总览、收益概览、资产配置、市场速览 |
| 持仓分析 | Portfolio | 持仓诊断、行业暴露、重叠检测、风险指标、调仓建议 |
| 基金池 | Funds | 基金搜索、综合评价、基金对比、历史走势 |
| 市场扫描 | MarketScan | 指数估值、行业热度、低估推荐、资金流向 |

## 二、页面详细功能

### 2.1 Dashboard（总览）

**数据指标**
- 总资产：所有持仓市值总和
- 总收益/收益率：持仓盈亏
- 持有基金数
- 市场估值分位：主要指数 PE 历史百分位均值

**可视化**
- 资产配置饼图（股票/债券/商品/货币比例）
- 主要指数估值表（PE/PB 及历史百分位）
- 持仓基金列表（代码、名称、份额、成本、收益）

**交互**
- 点击基金跳转至基金详情
- 估值百分位颜色标识（绿<30% / 黄30-60% / 红>60%）

### 2.2 Portfolio（持仓分析）

**风险指标**
- 年化波动率
- 最大回撤
- 夏普比率
- VaR(95%) 日风险价值

**分析维度**
- 行业暴露：持仓穿透到底层股票的行业分布，横向柱状图
- 持仓重叠检测：不同基金共同持有同一股票的风险
- 基金相关性与持仓集中度

**输出**
- 调仓建议（TODO：基于目标配置的再平衡通知）

### 2.3 Funds（基金池）

**基金搜索**
- 通过基金代码或名称关键词搜索
- 支持模糊匹配

**基金评价**
- 综合评分（0-100）
- 收益同类排名百分位
- 风险等级
- 夏普比率
- 最大回撤
- 综合费率

**拓展功能（待实现）**
- 多基金对比
- 历史净值走势图
- 基金经理表现追踪
- 基金持仓穿透

### 2.4 MarketScan（市场扫描）

**指数估值**
- 主要宽基指数（沪深300、中证500、创业板、恒生、标普500）
- PE / PB / 股息率
- PE/PB 历史百分位（自动判断高估/低估）

**低估机会**
- 推荐当前处于历史低估值区间的指数
- 附推荐理由

**行业热力图**
- 主要行业的涨跌幅监控
- 动量判断

**资金流向**
- 北向资金（陆股通）每日净流入/流出
- 柱状图展示近期趋势

## 三、数据模型

### 持仓数据
```
持有基金代码 | 基金名称 | 持有份额 | 持仓成本 | 当前市值 | 账户
```

### 基金数据
```
代码 | 名称 | 类型 | 基金经理 | 基金公司 | 成立日期 | 净值
```

### 净值历史
```
基金代码 | 日期 | 单位净值 | 累计净值
```

### 指数估值
```
指数代码 | 指数名称 | PE | PB | 股息率 | PE百分位 | PB百分位
```

## 四、数据来源（规划）

| 数据类型 | 优先数据源 | 备用数据源 |
|---------|-----------|-----------|
| 基金净值 | akshare (fund_etf_hist_em) | 天天基金网 |
| 基金持仓 | akshare (fund_portfolio_hold) | 巨潮资讯网 |
| 指数估值 | akshare (stock_zh_index_valuation) | 中证指数官网 |
| 北向资金 | akshare (stock_hsgt_north_net_flow_in_em) | 东方财富 |
| 行业板块 | akshare (stock_board_industry_hist_em) | 同花顺 |

## 五、业务规则

### 估值判断标准
- PE/PB 百分位 <= 30%：低估（绿色），关注加仓
- PE/PB 百分位 30%-60%：中等（黄色），持有观察
- PE/PB 百分位 > 60%：高估（红色），考虑减仓

### 风险预警规则（待实现）
- 单行业集中度 > 25%：预警
- 单只基金占比 > 20%：预警
- 持仓相关性均值 > 0.7：预警
- 最大回撤超过用户风险承受能力：预警
